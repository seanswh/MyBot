## 第2页：产品概述

**定义**：

**部署形态**：
- CLI工具链
- Gateway网关服务
- 桌面应用（macOS）
- 移动节点（iOS/Android）

## 第5页：实战案例 - 地震监控系统

### **项目背景**
**用户需求**：实时获取全球地震信息通知  
**解决方案**：基于Wolfx.jp API的自动化监控系统

### **技术实现**
```python
# earthquake_check.py 核心逻辑
1. 每10分钟调用API：https://api.wolfx.jp/cenc_eqlist.json
2. 对比上次状态：earthquake_state.json
3. 检测新地震：EventID对比
4. 格式化输出：中文友好格式
5. 通知用户：通过消息通道
```

### **技能封装**
```
# SKILL.md 结构
---
name: earthquake-query
description: 查询Wolfx.jp地震API，获取最新地震信息
metadata: {"openclaw": {"emoji": "🌍"}}
---

## 使用方法
- 直接查询API：web_fetch工具
- 运行脚本：python3 earthquake_check.py
- 数据分析：JSON解析和格式化
```

### **系统运行效果**
- ✅ 24/7自动监控
- ✅ 增量检测避免重复
- ✅ 多格式输出支持
- ✅ 状态持久化存储

## 第6页：开发体验与工作流

### **快速启动流程**
```bash
# 1. 安装
curl -fsSL https://openclaw.ai/install.sh | bash

# 2. 配置向导
openclaw onboard --install-daemon

# 3. 连接消息通道
openclaw channels login  # WhatsApp扫码

# 4. 测试
openclaw message send --target +15555550123 --message "测试"
```

### **技能开发流程**
**四步开发法**：
1. **分析需求**：明确功能边界
2. **编写脚本**：Python/Shell等实现
3. **创建技能**：编写SKILL.md文档
4. **测试部署**：本地验证 → 发布

### **配置管理**
```json
// ~/.openclaw/openclaw.json
{
  "channels": {
    "whatsapp": {
      "allowFrom": ["+8618701011974"],
      "groups": { "*": { "requireMention": true } }
    }
  },
  "messages": {
    "groupChat": { "mentionPatterns": ["@openclaw"] }
  }
}
```

### **定时任务系统**
```yaml
# cron配置示例
定时任务：每10分钟检查地震
schedule: { "kind": "every", "everyMs": 600000 }
payload: { "kind": "systemEvent", "text": "执行地震检查" }
sessionTarget: "main"
```

## 第8页：安全架构

### **网络模型设计**
- **Loopback优先**：本地通信默认限制
- **Token认证**：网关访问令牌系统
- **端口绑定策略**：
  - WebSocket: 18789 (控制平面)
  - HTTP: 18793 (Canvas宿主)

### **访问控制层级**
1. **通道级**：allowFrom白名单
2. **群组级**：requireMention配置
3. **会话级**：配对码批准机制
4. **代理级**：沙箱隔离策略

### **数据安全设计**
- **工作区隔离**：不同用户/用途独立环境
- **会话分离**：群组与私聊独立处理
- **凭据管理**：OAuth和API密钥安全存储
